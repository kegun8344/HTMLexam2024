GIT versjonskontrollsystem

Git er et versjonskontrollsystem, som er et program som holder orden på nye og tidligere filer. Filene slettes ikke, men lagres i en database. Vi bruker GitHub. Dette gjør det mulig å spore filer, eller endringer i kodeprosjektet vårt, og generelt øker orden på prosjektet. Git gjør det mulig å enkelt gå tilbake til tidligere versjoner av en kode, og se forskjellen det utgjør på nettsiden.

Bruk av Pull Request og Merge

Når en branch er ferdig, sendes den over til Github, og der kan andre brukere på projektet se koden. Dette er for å foreslå endringer, uten å direkte legge koden inn i projektet. Pull requests øker samarbeid mellom brukere, og gjør det lettere å forbedre koden til selve projektet. Merge er når den foreslåtte koden blir godkjent, og er klar for å integreres til 'main'. Nå er den utviklede branchen ferdig.

Arbeid samtidig på samme projekt

Git lar brukere lage det som heter "branches", som er separate filer fra hovedprojektet. Hovedbranchen kalles oftest for "main", og dette er her vi danner branches fra. Uten branches ville samarbeid på et projekt vær utfordrende, og forstyrrende. Når en branch er ferdig, kan den testest eller sjekkes, og integreres til 'main'.

Github gjør mulighet for Sporing av Bidrag

Github registerer aktiviteten til brukere, og ser hvem som har gjort hva. Dette gir oss mulighet til å få oversikt over hvem som har gjort hvilken del av projektet.
